# 批次 1 — 「資料更正」回報處理 SOP

> 收到一筆「資料更正」回報後，照這份做。涵蓋三小類：親屬關係、名字、生日。
> 一切遵循 `CLAUDE.md`：`wiki/*.md` 是唯一正本，回報只是提示，校訂後才寫入。

---

## 步驟 0：定位是哪一隻

- 從個體頁按鈕來的回報，收件匣會有 **`panda`／`slug`／`url`** 三欄自動帶入身分——**直接用 `slug` 欄定位 wiki 檔案**（最精準），`url` 可點開該頁核對。
- 萬一是少數沒帶參數的回報（例如直接開 `/feedback`），就靠描述中的名字，用 `python tools/query.py profile <名>` 找；同名多隻再靠生日／動物園／父母縮小。對不到就先擱置，必要時回問回報者。

## 步驟 1：紅綠燈分級

| 燈號 | 條件 | 動作 |
|------|------|------|
| 🟢 綠 | 有可信來源（官網／公告／新聞／可查證 RPF），且只是補空白或修正明顯錯字 | 直接進步驟 2 起草 |
| 🟡 黃 | 與 wiki 現有資料衝突，或來源是「我記得／沒附連結」 | 先比對 RPF／lineage／官網補佐證，再升🟢或降🔴；查不到就暫不改 |
| 🔴 紅 | 無來源又想覆蓋既有校訂（尤其中文名／暱稱／生日），或明顯灌水／與多源矛盾 | 收件匣標「退回＋原因」，**不動 wiki** |

> 口訣：綠燈補空白、黃燈先查證、紅燈守正本。

## 步驟 2：依小類處理（Claude 起草 → 你同意 → 寫入）

### A. 親屬關係（例：不是兄弟，是雙胞胎）

1. 用 `python tools/query.py profile <兩隻>` 比對：**雙胞胎 = 同父母且同生日**；同年不同日只是手足。
2. 改兩隻條目的 `## 家族關係` 區塊文字，標明關係（雙胞胎用「雙胞胎」字樣；½ 表半血緣）。
3. 確認彼此用 `[[slug]]` 互連（對方有條目才連）。
4. 重建：`python3 tools/build_db.py` → `python3 pipeline/scripts/export_json.py`。

### B. 名字（拼法／中文名／暱稱）

1. 中文名／暱稱**以作者校訂為準**，回報需有來源才採（RPF 羅馬拼音僅後備）。
2. 改該條目 frontmatter 對應欄位（`name`／`chinese`／`japanese`／`nicknames`／`english_variants`）與 `## 名稱與別名` 表。RPF 的 Other Names 記得歸到 `japanese`／`english_variants`。
3. 重建：`build_db.py` → `export_json.py`。

### C. 生日 ⚠️ 連鎖效應最大

1. **務必有可靠來源**才動；否則維持🟡暫不改。
2. 改 frontmatter `born:`。
3. **改檔名**為 `slug-新生日.md`（slug 規則見 `CLAUDE.md`「檔名與消歧」）。
4. **全庫搜尋舊 slug，把所有 `[[舊slug]]` 換成 `[[新slug]]`**。
5. 若 `zoos:` 用了年份起訖、或撞名消歧用到生日，一併檢查。
6. 重建：`python3 tools/gen_residence.py` → `build_db.py` → `export_json.py`。

## 步驟 3：記 log

在 `wiki/log.md` 末端 append 一筆 `update`，來源寫回報依據的連結。
**log 內名字一律用 backtick（`` `yuuta` ``），絕不用 `[[wikilink]]`。**

```markdown
## [YYYY-MM-DD] update | 依回報更正 `名字` 的<關係／名字／生日>

**來源**：
- <回報附的官網／RPF 連結>

**更新條目**：
- `slug.md` — <改了什麼>
```

## 步驟 4：驗收與收尾

- `python tools/audit.py` 檢查完整度（缺欄位／與 lineage 不符提示）。
- 條目總數用 `ls wiki/*.md | wc -l` 減 2 核對（若有增減）。
- 在收件匣（Google Sheet）把該筆標為「已處理」，要致謝就記下暱稱。
- 推 `main` → GitHub Actions 自動部署。

---

## 退回回報怎麼說（紅燈／資料不足）

語氣友善、說明資料以作者校訂為準。例：
> 謝謝回報！這筆我們目前查不到可佐證的來源，與現有校訂資料也不一致，所以暫時保留。如果你有動物園官網或公告連結，再麻煩補上，我們會再評估 🙏
