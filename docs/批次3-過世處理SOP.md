# 批次 3 —「過世 / 出生」回報處理 SOP

> 收到一筆「🌈 過世」或「🍼 出生」回報後，照這份做。
> 一切遵循 `CLAUDE.md`：`wiki/*.md` 是唯一正本，回報只是提示，校訂後才寫入。
> 過世見以下；出生見文末〈出生〉。

---

## 這類回報長什麼樣

從個體頁「✏️ 回報這頁資料／近況」按鈕來，收件匣有 `panda`／`slug`／`url` 帶入身分。表單第 1 題（下拉）選「🌈 過世」時，用條件邏輯只顯示：

- **過世日期**（過世時必填，大概日期也可——只知月份就寫到月）
- **來源**（官網／可靠社群／新聞／IG／現場＋日期）

> 「正確的應該是什麼？」更正描述會被隱藏，過世回報只需填日期＋來源。

## 步驟 0：定位

用收件匣 `slug` 欄定位 `wiki/<slug>.md`。

## 步驟 1：紅綠燈

| 燈號 | 條件 | 動作 |
|------|------|------|
| 🟢 綠 | 訃聞有可信來源（官網／官方社群／新聞），日期明確或大概 | 進步驟 2 |
| 🟡 黃 | 來源是「聽說／沒附連結」，或日期完全不明 | 補查官網／可靠社群再決定；查不到暫不改 |
| 🔴 紅 | 無來源、明顯灌水，或與多源矛盾 | 收件匣標「退回＋原因」，**不動 wiki** |

> 過世是敏感且不可逆的標記，沒有可信來源**寧可維持🟡**，別輕易標一隻還在世的熊為已故。

## 步驟 2：改 frontmatter

1. 加 `died: YYYY-MM-DD`（只知月份就 `YYYY-MM`，只知年就 `YYYY`）。
2. `tags:` 加 `deceased`。
3. 名稱不用手改——🌈 由網站依 `died` 自動呈現（見步驟 4）。

## 步驟 3：🌈 標記（依現有慣例）

- 網站會自動處理：設了 `died` 後，個體頁標題、搜尋結果、家系圖、以及**其他條目指向這隻的 `[[wikilink]]`** 都會自動加 🌈（`web` 的 `link()`／`displayName` 依 `died` 判斷）。
- wiki 內文若有手寫提及該個體名稱的地方，依慣例在名字後補 🌈（`log.md` 例外，一律用 backtick、不加 wikilink）。

## 步驟 4：重建管線

```bash
python3 tools/gen_residence.py     # died 會影響居住史「終」與現居呈現
python3 tools/build_db.py
python3 pipeline/scripts/export_json.py
```

## 步驟 5：記 log

`wiki/log.md` 末端 append 一筆 `update`，名字用 backtick、**不用 `[[wikilink]]`**：

```markdown
## [YYYY-MM-DD] update | 依回報標記 `名字` 過世 🌈

**來源**：
- <回報附的官網／訃聞連結>

**更新條目**：
- `slug.md` — 加 died: YYYY-MM-DD、tags +deceased
```

## 步驟 6：驗收

- `python tools/audit.py` 檢查完整度。
- 開該個體頁與首頁，確認名字旁出現 🌈、基本資料列出「過世日」。
- 收件匣標「已處理」，要致謝記下暱稱。
- 推 `main` → 自動部署。

---

## 退回回報怎麼說（無來源／日期不明）

> 謝謝回報。過世是很重要的紀錄，我們想確認可信來源（動物園官網或公告最好）與大概日期後再更新，以免標錯。若方便再麻煩補上，謝謝 🙏

# 出生

> 出生回報**掛在親代的個體頁**（回報者從媽媽／爸爸的頁面按按鈕）。收件匣的 `slug` 帶的是**那位親代**，不是寶寶。表單第 1 題選「🍼 出生」時，條件顯示：寶寶生日（必填）、補充（另一位親代／同胎幾隻／寶寶名字，選填）、來源。
>
> ⚠️ **範圍**：本表單只收「已知親代」的出生。父母不詳、或海外（如中國）無血統紀錄的個體，**不開放群眾回報**，由作者自行手建（需人工判斷）。

## 步驟 0：定位親代

`slug` 欄 = 親代的 wiki 檔。這位親代就是寶寶的父或母之一；另一位看「補充」欄（常是文字或簡稱，可用 RPF／既有條目比對，比對不到先留待確認）。

## 步驟 1：紅綠燈

| 燈號 | 條件 | 動作 |
|------|------|------|
| 🟢 綠 | 出生有可信來源（官網／官方社群／新聞），生日明確或大概 | 進步驟 2 建檔 |
| 🟡 黃 | 來源薄弱、生日不明、或另一位親代對不上 | 補查 RPF／官網；查不到先擱置 |
| 🔴 紅 | 無來源／灌水 | 收件匣標退回，不建檔 |

## 步驟 2：建檔（走 `CLAUDE.md`「新增成員流程」）

1. 建 `wiki/<寶寶名-生日>.md`；**未命名**就用佔位名＋媽媽名當 slug（見 `CLAUDE.md`「檔名與消歧」），待正式命名再改 slug。
2. frontmatter：`born:`（大概日期就寫到月／年）、`sex`（不確定留空備註待確認）、`zoos:`（多半＝親代現居園）、父母（一位是步驟 0 的親代；另一位填解析到的 canonical 名或留空待確認）、`species`、`sources`（回報附的連結）。
3. **同胎多隻**（補充欄有寫）→ 一隻一檔。
4. **自動補齊直系親屬**：父、母、雙胞胎、子女若無條目一律建立（順序：寶寶→父→母→雙胞胎）。
5. 回頭把相關既有條目的親屬純文字改成 `[[wikilink]]`。

## 步驟 3：更新索引與日誌

- `wiki/index.md`：加入適當家族分類、更新頁首「最後更新」與「條目總數」。
- `wiki/log.md` 末端 append `add`（名字用 backtick、**不用 `[[wikilink]]`**）。

## 步驟 4：重建與驗收

```bash
python3 tools/build_db.py
python tools/audit.py          # 檢查缺欄位／與 lineage 不符
python3 pipeline/scripts/export_json.py
```

條目總數用 `ls wiki/*.md | wc -l` 減 2 核對；推 `main` 自動部署。
